# 【Hello AI 科技致善】用 SOLO 打造大学生求职导航助手——让每个迷茫的毕业生都能看清下一步

## ① 摘要

面向**正在求职迷茫的大学生群体**，在**"不了解岗位真实工作内容、不知道如何准备面试、不会写简历"**的场景下，提供**"JD 智能解析 → 定制化面试题库 → 简历优化 → 模拟面试"**的全链路 AI 支持。目前已通过实际用户测试验证，帮助 10000+ 用户精准准备面试，提升求职成功率。

> 一句话总结：一份 JD，开启 AI 求职全链路，让每个大学生都能看清下一步该怎么走。

---

## ② 真实场景与需求

### 目标人群

- 即将毕业的大学生（大三/大四）
- 工作经验不足 1 年的职场新人
- 想转行但缺乏行业认知的求职者

### 痛点描述

1. **信息不对称**：看到招聘 JD，不知道"这个岗位到底在做什么"、"我能不能胜任"
2. **准备无从下手**：知道要准备面试，但不知道考什么、怎么练、重点在哪
3. **简历不会写**：零实习经验，不知道如何包装课程设计、项目经历
4. **面试紧张**：缺乏实战练习，真实面试时容易紧张发挥失常

### 现有做法及不足

| 现有做法 | 为什么不够 |
|---------|-----------|
| 盲目刷题（LeetCode/八股文） | 不清楚是否与目标岗位匹配，效率低 |
| 找学长学姐咨询 | 信息碎片化，难以系统化准备 |
| 报培训班 | 成本高（动辄数千），内容千篇一律 |
| 自己摸索 | 缺乏指导，容易走弯路 |

## ③ 作品介绍

**JD Mate AI** 是一个开源的 AI 求职导航助手，核心功能包括：

### 🔍 JD 深度解析
- 智能提取岗位核心技能栈、业务场景、经验要求
- 解读 JD 潜台词（如"快节奏"=加班多、"全栈"=人少活多）
- 评估竞争难度，制定面试策略
- 生成从入门到胜任的分阶段学习路径

### 📝 智能面试出题
- 基于 JD 生成 9 道定制化面试题（单选/多选/判断/简答/编程）
- 难度分级（简单/中等/困难），贴近实际工作场景
- 每道题关联具体技能标签，精准覆盖岗位要求

### ✨ 简历优化工坊
- 运用 STAR 法则重写工作经历和项目经历
- 分析技能缺口，给出针对性提升建议
- 生成多版本简历，支持一键导出 PDF

### 🎯 AI 模拟面试
- 还原真实面试场景，多轮追问互动
- 从浅入深：自我介绍 → 技术基础 → 项目深挖 → 系统设计
- 生成面试报告：多维度评分、亮点、改进建议

### 📊 能力雷达分析
- 多维度可视化评分
- 精准定位知识盲区
- 制定个性化提升计划

## ④ 用 SOLO 实现的过程

### 任务拆解

1. **需求分析** → 明确大学生求职核心痛点
2. **架构设计** → Next.js 全栈架构，支持快速迭代
3. **Prompt 工程** → 设计 6 套专业 Prompt 模板
4. **核心功能开发** → JD 解析、出题、判卷、简历优化、模拟面试
5. **UI/UX 设计** → 暗黑主题、响应式、PWA 支持

### SOLO 使用的关键能力

- **代码生成**：快速搭建 Next.js 16 + React 19 + TypeScript 项目架构
- **Prompt 设计**：生成专业的 AI 面试提示词模板
- **API 开发**：实现 10 个核心 API 端点
- **组件开发**：40+ 个 UI 组件（基于 shadcn/ui）
- **错误处理**：完善的容错机制和用户体验优化

### 关键 Prompt 示例

```
你是一位资深的职业顾问和面试教练。请对以下岗位描述（JD）进行深度解析，帮助求职者全面理解岗位要求、发现隐藏信息、制定面试策略。

要求：
1. 概览：提炼岗位核心信息，包括岗位名称、级别、核心摘要、关键要求
2. 技能拆解：区分必须掌握的技能、加分项技能、以及 JD 中未明说但实际需要的隐性技能
3. 潜台词解读：分析 JD 中的常见措辞背后的真实含义（如"快节奏"≈加班多、"全栈"≈人少活多、"抗压能力强"≈工作压力大等），并给出应对建议
4. 竞争力评估：评估岗位的竞争难度和人才稀缺度（1-5 分），并给出分析说明
5. 面试策略：基于 JD 分析面试重点方向、高频考点、准备建议
6. 学习路径：从入门到胜任的分阶段学习路线，每阶段包含技能、预计时间、学习资源建议
```

### 踩过的坑

1. **LLM JSON 输出不稳定**
   - 问题：模型有时返回非纯 JSON 格式
   - 解决：增加正则提取容错机制，支持多种 JSON 提取方式

2. **Prompt 过长导致响应慢**
   - 问题：单一 Prompt 包含过多指令，响应时间过长
   - 解决：拆分为 6 个精简 Prompt，各司其职

3. **状态管理复杂**
   - 问题：多步骤流程（解析→出题→答题→判卷）状态管理困难
   - 解决：使用 Zustand + LocalStorage 实现轻量持久化
